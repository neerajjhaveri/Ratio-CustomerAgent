3 ── METRIC COMPUTATION MIND MAP
═══════════════════════════════════════════════════════════════════

METRIC FORMULAS:

  AIRO (default)
    +-- Formula: SUM(ImpactedResource) / NULLIF(MAX(AR_*), 0)
    +-- Field names by slice:
    |     Fleet:    AIRO_Fleet     (basis: AR_Fleet)
    |     Region:   AIRO_Region    (basis: AR_Region)
    |     Customer: AIRO_Customer  (basis: AR_Customer)

  AIROD (only if explicitly requested)
    +-- Formula: (SUM(ImpactedResource) * SUM(ImpactDuration)) / NULLIF(MAX(AR_*), 0)
    +-- Field names: AIROD_Fleet, AIROD_Region, AIROD_Customer

  SLI Variants (only if explicitly requested)
    +-- Swap: ImpactedResource -> ImpactedResourceSLI
    +--        ImpactDuration  -> ImpactDurationSLI
    +-- Never mix ACM and SLI components in one metric

COMPUTATION SOURCE:
  +-- ComputationSource = CASE WHEN CommunicatedIncidentId IS NOT NULL
                               THEN 'ACM' ELSE 'SLI' END

CONTRIBUTION PERCENTAGE (Region/Customer slices only):
  +-- ContributionPct = 100.0 * SUM(ImpactedResource_slice)
                        / NULLIF(SUM(ImpactedResource_offering_total), 0)
  +-- Compute via window function over (Offering, IncidentId)

NUMERIC SAFETY:
  +-- Multiply numerator by 1.0 or explicit CAST to avoid integer division
  +-- Use NULLIF for ALL denominators
  +-- Exclude rows where AR_* is zero (note in explanations if excluded)
  +-- ROUND metrics to 4 decimal places in SQL
  +-- Cast final metrics to DECIMAL(10,4)
  +-- Narrative may show 2 decimal places

5 ── QUERY CONSTRUCTION MIND MAP
═══════════════════════════════════════════════════════════════════

CTE CHAIN PATTERN (build in this order):

  FleetAgg
    +-- GROUP BY IncidentId, Offering
    +-- Compute AIRO_Fleet (+ AIROD_Fleet if requested)
    |
    v
  TopOfferings
    +-- ROW_NUMBER() PARTITION BY IncidentId ORDER BY AIRO_Fleet DESC
    +-- Filter: RowNum <= N (default 3)
    |
    v
  RegionAgg (if Region slice needed)
    +-- JOIN TopOfferings to filter only top offerings
    +-- GROUP BY IncidentId, Offering, Region
    +-- Compute AIRO_Region + ContributionPct
    |
    v
  RegionRanked
    +-- ROW_NUMBER() PARTITION BY IncidentId, Offering
    |   ORDER BY AIRO_Region DESC
    +-- Filter: RowNum <= 10 (or user override)
    |
    v
  CustomerAgg (if Customer slice triggered)
    +-- JOIN TopOfferings to filter only top offerings
    +-- GROUP BY IncidentId, Offering, CustomerName
    +-- Compute AIRO_Customer + ContributionPct
    |
    v
  CustomerRanked
    +-- ROW_NUMBER() PARTITION BY IncidentId, Offering
    |   ORDER BY AIRO_Customer DESC
    +-- Filter: RowNum <= 10 (or user override)
    |
    v
  HourlyAgg (if trend triggered)
    +-- GROUP BY IncidentId, Offering, HourBucket

SAMPLE QUERIES:

Fleet:
  SELECT IncidentId, Offering,
    CASE WHEN CommunicatedIncidentId IS NOT NULL THEN 'ACM' ELSE 'SLI' END AS ComputationSource,
    MAX(AR_Fleet) AS ActiveResourceFleetCount,
    SUM(ImpactedResource) AS ImpactedResourceCount,
    CAST(ROUND(1.0 * SUM(ImpactedResource) / NULLIF(MAX(AR_Fleet),0), 4) AS DECIMAL(12,4)) AS AIRO_Fleet
  FROM dbo.OfferingOutageAiro
  WHERE IncidentId = @IncidentId
  GROUP BY IncidentId, Offering, CommunicatedIncidentId
  ORDER BY AIRO_Fleet DESC;

Region:
  SELECT IncidentId, Offering, Region,
    CASE WHEN CommunicatedIncidentId IS NOT NULL THEN 'ACM' ELSE 'SLI' END AS ComputationSource,
    MAX(AR_Region) AS ActiveResourceRegionCount,
    SUM(ImpactedResource) AS ImpactedResourceCount,
    CAST(ROUND(1.0 * SUM(ImpactedResource) / NULLIF(MAX(AR_Region),0), 4) AS DECIMAL(12,4)) AS AIRO_Region
  FROM dbo.OfferingOutageAiro
  WHERE IncidentId = @IncidentId
  GROUP BY IncidentId, Offering, Region, CommunicatedIncidentId
  ORDER BY AIRO_Region DESC;

Customer:
  SELECT IncidentId, Offering, CustomerName,
    CASE WHEN CommunicatedIncidentId IS NOT NULL THEN 'ACM' ELSE 'SLI' END AS ComputationSource,
    MAX(AR_Customer) AS ActiveResourceCustomerCount,
    SUM(ImpactedResource) AS ImpactedResourceCount,
    CAST(ROUND(1.0 * SUM(ImpactedResource) / NULLIF(MAX(AR_Customer),0), 4) AS DECIMAL(12,4)) AS AIRO_Customer
  FROM dbo.OfferingOutageAiro
  WHERE IncidentId = @IncidentId
  GROUP BY IncidentId, Offering, CustomerName, CommunicatedIncidentId
  ORDER BY AIRO_Customer DESC;

7 ── HOURLY CROSS-SERVICE IMPACT
═══════════════════════════════════════════════════════════════════

Trigger: "other services", "other offerings", "hourly trend", "trend",
"first hours", "early impact"

SCHEMA ALIGNMENT / IMPORTANT NOTE
OfferingAirodTrend does NOT contain IncidentId. Hourly impact is inferred by:
1) Identifying impacted services for the Incident via
   Factless_IncidentImpactedServices (ResponsibleIncidentId = @IncidentId,
   IsQCService = 1).
2) Applying the Incident's global time window
   (Fact_Outage.ImpactStartDate -2h .. MitigateDate/+10h)
   to hourly rows in OfferingAirodTrend filtered by ServiceName.

DATA SOURCES:
- OfferingAirodTrend: hourly AIRO basis data
  (CalendarDayHour, ServiceName, Offering, Region, CustomerName,
   AR_Fleet, AR_Region, ImpactedResourceCount, ImpactDuration)
- Factless_IncidentImpactedServices: impacted services list
  (ResponsibleIncidentId, ImpactedService, IsQCService)
- Fact_Outage: Incident-level ImpactStartDate & MitigateDate (time window anchor)

TIME WINDOW (FACT_OUTAGE ANCHORED):
1. Get ImpactStartDate, MitigateDate for @IncidentId.
2. StartHour = DATEADD(hour, -2, ImpactStartDate).
3. NominalEnd = DATEADD(hour, +10, ImpactStartDate).
4. EndHour = CASE WHEN MitigateDate IS NOT NULL AND MitigateDate < NominalEnd
              THEN MitigateDate ELSE NominalEnd END.
5. Filter OfferingAirodTrend WHERE CalendarDayHour BETWEEN StartHour AND EndHour.

MULTI-CRITERION SERVICE SELECTION (for top-N services):
1. Compute per-service aggregates inside window:
   CumImpactedResource, CumAIRO, PeakHourAIRO, PeakHourImpactedResource,
   PositiveImpactHours.
2. Build three ranked lists (default K=5):
   Intensity (CumAIRO), Peak (PeakHourAIRO), Volume (CumImpactedResource).
3. Union lists + requested service.
4. Add high-ImpactScore services:
   ImpactScore = 0.4*N_CumAIRO + 0.3*N_PeakAIRO + 0.2*N_CumImpRes + 0.1*N_PeakImpRes
5. Auto-include services with PeakHourAIRO >= 0.12 (threshold).
Defaults: @MaxOtherServices = 5 (hard cap 15).

OUTPUT FOR HOURLY:
Fields: ServiceName, Offering, CalendarDayHour, ActiveResourceFleetCount,
        ImpactedResourceCount, AIRO_Hourly (DECIMAL(10,4))
Order: Requested service rows first (chronological), then others by
       ListAppearances DESC, ImpactScore DESC, ServiceName ASC.
