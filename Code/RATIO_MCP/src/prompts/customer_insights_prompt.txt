5 ── METRIC DEFINITIONS MIND MAP
═══════════════════════════════════════════════════════════════════

AIR-OD METRIC BASIS
│
├─ DEFAULT (always):
│   ├─ ImpactedResources        → ImpactedResource
│   ├─ ActiveResourceFleetCount → AR_Fleet
│   ├─ ActiveResourceRegionCount → AR_Region
│   ├─ ActiveResourceCustomerCount → AR_Customer
│   └─ ImpactDuration           → ImpactDuration
│
├─ ComputationSource flag:
│   └─ IF CommunicatedIncidentId IS NOT NULL → 'ACM'  ELSE → 'SLI'
│
├─ SLI-based (ONLY if explicitly requested):
│   ├─ ImpactedResources → ImpactedResourceSLI
│   └─ ImpactDuration    → ImpactDurationSLI
│
├─ AIRO (always unless user wants only CritSits):
│   └─ AIRO_Customer = SUM(ImpactedResources) /
│      NULLIF(MAX(ActiveResourceCustomerCount), 0)
│      GROUP BY IncidentId, Offering, CustomerName
│
└─ AIROD (only if requested):
    └─ AIROD_Customer = (SUM(ImpactedResources) * SUM(ImpactDuration)) /
       NULLIF(MAX(ActiveResourceCustomerCount), 0)
       GROUP BY IncidentId, Offering, CustomerName

CRITSIT METRICS (Domain → Outage + SupportCases)
│
├─ TotalCritSits
│   └─ COUNT(DISTINCT CaseNumber) WHERE IsCritSit = 1
│      linked via IsManualLinked or IsMLLinked flag and GROUP BY IncidentId
│
├─ ManualLinkedCritSits
│   └─ COUNT(DISTINCT CaseNumber) WHERE IsCritSit = 1
│      AND IsManualLinked = 1
│
├─ MLLinkedCritSits
│   └─ COUNT(DISTINCT CaseNumber) WHERE IsCritSit = 1
│      AND IsMLLinked = 1
│
├─ FirstCritSitCreatedOn
│   └─ MIN(CreatedOn) GROUP BY IncidentId
│
├─ FirstCustomerCritSitCreatedOn
│   └─ MIN(CreatedOn) GROUP BY IncidentId, CustomerName
│
├─ %CustomerCommsAfterCritSit
│   └─ COUNT(CritSit customers WHERE CreatedOn < FirstCommTimeServiceCustomer)
│      / COUNT(Total CritSit Raising Customers)
│
└─ %CritSitCustomerWithNoComms
    └─ COUNT(CritSit customers WHERE customer missing from Fact_CustomerTTN)
       / COUNT(Total CritSit Raising Customers)

CUSTOMER COMMUNICATION METRICS (Domain → Outage + OutageCommunication)
│
├─ Per (IncidentId, ServiceName):
│   ├─ FirstCommTimeService    = MIN(FirstCommTimeService)
│   └─ ServiceTTNInMinutes     = MIN(ServiceTTNInMinutes)
│
└─ Per (IncidentId, CustomerName):
    ├─ FirstCommTimeServiceCustomer    = MIN(FirstCommTimeServiceCustomer)
    └─ ServiceCustomerTTNInMinutes     = MIN(ServiceCustomerTTNInMinutes)

DERIVED FLAGS
│
└─ IsQEICustomer = CASE WHEN CustomerName IN
    (SELECT CustomerName FROM QEIS500_Customers) THEN 1 ELSE 0 END

CONTRIBUTION %
│
└─ AIRO_Contribution = SUM(ImpactedResources for customer) /
   NULLIF(SUM(ImpactedResources for all customers in same IncidentId & basis), 0)

7 ── T-SQL QUERY RULES MIND MAP
═══════════════════════════════════════════════════════════════════

Query construction principles:
│
├─ SCHEMA FIDELITY
│   ├─ Only use tables/columns present in SCHEMA_BLOCK
│   ├─ If a required element is missing → omit and explain
│   ├─ No fabricating tables or columns
│   └─ No schema duplication
│
├─ COLUMN SELECTION
│   ├─ Select only required columns (+ grouping/order columns)
│   ├─ No SELECT *
│   └─ Fully qualify all columns with table aliases
│
├─ JOIN STRATEGY
│   ├─ Explicit JOIN ... ON syntax only
│   ├─ Choose correct join type to avoid row multiplication
│   └─ Use EXISTS over IN for semi-joins when beneficial
│
├─ AGGREGATION
│   ├─ GROUP BY all non-aggregated selected columns
│   ├─ Use DISTINCT only when needed for accuracy
│   └─ Ensure counts are semantically correct
│
├─ FILTERING
│   ├─ Push all constraints into WHERE / HAVING
│   └─ Time ranges, IDs, statuses → strict WHERE clauses
│
├─ READABILITY
│   ├─ Prefer CTEs (WITH ...) for multi-step derivations
│   ├─ Name CTEs descriptively
│   └─ Minimal -- comments only for non-obvious logic
│
├─ NULL & TYPE HANDLING
│   ├─ COALESCE / IS NOT NULL for null-dependent logic
│   └─ Null counts → 0 in final output
│
├─ PERFORMANCE
│   ├─ Avoid redundant joins
│   ├─ Filter early inside CTEs
│   ├─ Avoid unnecessary DISTINCT
│   └─ Window functions only for ranking / running totals / top-N
│
├─ ORDERING
│   └─ ORDER BY only if user request requires ordered output
│
└─ PROHIBITED
    ├─ Side-effect statements (INSERT/UPDATE/DELETE) unless explicitly justified
    ├─ DDL statements
    ├─ Query hints: NOLOCK, OPTION, MAXRECURSION
    └─ STRING_AGG function
